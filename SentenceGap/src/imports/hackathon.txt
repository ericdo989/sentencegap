Core pitch

SentenceGap exposes sentencing inequality by comparing similar criminal cases and showing how outcomes change across race, income, location, lawyer access, and charge type.

Tagline:

Same charge. Same history. Different sentence.

Best prize targets

Go for:

Use U.S. Sentencing Commission data because it has public federal sentencing datafiles, and the Commission says its individual datafiles contain case-level information on sentences imposed.

Tech stack

Use:

Frontend: React + Tailwind
Backend: Flask or FastAPI
Model: scikit-learn regression/classification
Database: MongoDB Atlas
AI: Gemini or Claude API
Charts: Recharts
Map: optional, only if time
Model plan

Use:

Linear Regression or Random Forest for sentence length prediction
Logistic Regression for incarceration vs non-incarceration
Statistical tests for disparity:
t-test
ANOVA
regression coefficients
R² / MAE

This helps you qualify for Best Statistical Model.

Dataset options

Best source:

U.S. Sentencing Commission individual offender datafiles for federal sentencing outcomes.

Backup/add-on:

Sentencing Project state-level criminal justice data for broad disparity context.
Stanford Open Policing Project if you want a policing/disparity comparison, though it is stop data, not sentencing data.

Demo script

Say this:

SentenceGap is a criminal justice transparency tool that shows how similar defendants can receive very different sentences. We trained a model on public federal sentencing data and built a simulator where users can change one factor at a time. The goal is not to replace lawyers or judges, but to expose patterns that are usually hidden inside court data.

Then demo:

Here are two similar cases. Same offense. Same criminal history. But when we change the district and demographic profile, the predicted sentence increases by 14 months. SentenceGap turns that hidden disparity into something visible.
